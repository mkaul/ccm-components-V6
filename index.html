Welcome to ccm-components pages.

ccm is a client side component model. The client is the browser, which loads web components on demand from any servers (maybe many, even cross domain). Every web component consists of CSS, JavaScript and JSON files. JavaScript is used to render HTML into div-s and span-s. iFrames are not necessary.

The Client-side Component Model (ccm) is a componentization model for the Web. It is based on the master thesis of Andr√© Kless, 2015, at the University of Applied Sciences Bonn-Rhein-Sieg introducing JavaScript+CSS-JSON-components which may be loaded cross-domain via JSONP and rendered into any div or span in any web page. For introductory tutorials see the Wiki Pages in his GitHub repository

W3C custom elements

W3C custom elements can easily be realized via ccm components e.g. as follows. 
            <!-- Call ccm component via W3C custom element -->
            <ccm-element data-component="clock"></ccm-element>
A tighter integration into ccm will follow soon: 
            <!-- Call ccm component via W3C custom element -->
            <ccm-clock data-config=""></ccm-clock>
